2. Why Prompt Engineering?

A better prompt produces a better answer without changing the model.

Good prompts specify: - Role - Task - Context - Constraints - Output
format

Example: “You are a Python teacher. Explain recursion with one example.”

3. PromptTemplate

Library: from langchain_core.prompts import PromptTemplate

Purpose: Creates reusable prompts using placeholders.

Instead of f-strings: f”Explain {topic}”

Use: PromptTemplate(template=“Explain {topic}”,
input_variables=[“topic”])

Advantages over f-strings - Input validation - Reusable - Cleaner code -
Easy integration with LangChain chains - Safer variable substitution

Important APIs - format() - invoke()

4. ChatPromptTemplate

Library: from langchain_core.prompts import ChatPromptTemplate

Unlike PromptTemplate, it creates chat messages instead of one text
block.

Supports roles: - system - human - ai

Example workflow: System -> Defines assistant behaviour Human -> User
question AI -> Previous response (optional)

5. Message Classes

Library: from langchain_core.messages import ( SystemMessage,
HumanMessage, AIMessage )

SystemMessage: Controls personality and rules.

HumanMessage: Represents user input.

AIMessage: Stores assistant responses and chat history.

6. MessagePlaceholder

Library: from langchain_core.prompts import MessagesPlaceholder

Purpose: Dynamically injects previous chat history into
ChatPromptTemplate.

Useful for: - Chatbots - Memory - Multi-turn conversations

Flow: Previous Messages ↓ MessagesPlaceholder ↓ Current Question ↓ LLM

7. invoke()

invoke(“hello”) Single string

invoke(messages) List of messages

Both return AIMessage.

8. Chat History

Typical workflow

SystemMessage ↓ HumanMessage ↓ AIMessage ↓ HumanMessage ↓ AIMessage

Appending AI responses keeps conversation context.

11. Common Errors Found

-   MessagePlaceholder should be MessagesPlaceholder.
-   HummanMessage should be HumanMessage.
-   chat_history.extend(f.read()) extends character by character.
    Better: chat_history = f.readlines() or parse saved messages.
-   Maintain chat history as message objects.
